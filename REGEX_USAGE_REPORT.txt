7. ASSESSMENT — WHAT TO REPLACE vs WHAT TO KEEP
================================================================================

SEMANTIC / LINGUISTIC (replace with learned semantic layer):
  - Intent/question-type classification tables:
      prefrontal_workspace.py (55), engine.py (62), question_decomposition.py (43),
      response_gen.py (23), chain_walker.py (10), interface.py (2)
      => ~195 sites. These hardcoded "what/why/how/compare/analogy/greeting"
         rules are the CORE of what a chat-learned semantic layer must replace.
  - Pragmatics & personal modeling:
      implicature_detector.py (4), user_model.py (8), conversational_repair.py (3),
      response_gen.py affect/self blocks (≈6)
      => ~21 sites. Social/affect/self-reference detection.
  - Relation / definition / quantity extraction from text:
      proposition_parser.py (7), relation_memory.py (2), quantity_modifier.py (4),
      web_learning.py definition-extraction patterns (≈12),

BOTTOM LINE:
  The repo leans HEAVILY on regex for language understanding — especially intent
  detection (5 files, ~195 sites) and query-type routing (question_decomposition.py,
  43 sites). Those ~255 semantic sites are exactly the surface a unified learned
  semantic layer should absorb. The ~78 structural/IO sites (HTML/JSON scraping,
  whitespace, corpus building, code reflection) are benign and should stay as regex.
  No third-party `regex` module is in use; all 333 sites use stdlib `re`.
  Recommended migration order: (1) question_decomposition.py + prefrontal_workspace.py
  + engine.py intent tables, (2) implicature/user_model/repair pragmatics,
  (3) proposition/relation/quantity extractors, (4) fold word/sentence tokenization
  into the layer's tokenizer last.
